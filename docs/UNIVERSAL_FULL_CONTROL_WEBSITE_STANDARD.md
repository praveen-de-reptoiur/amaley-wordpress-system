# Universal Full-Control Website Standard

Read this before any website, Elementor, plugin, widget, template, archive page, single page, layout, or UI build.

## Core rule

Every section and every element needs non-coder friendly controls for content, visibility, layout, style, and responsive behavior.

In simple language:

Har section, har element ka content, show/hide, layout, style, spacing, and responsive control hona chahiye.

## Scope

This standard applies to all future projects, not only Amaley. It applies to e-commerce, institutional, NGO, travel, directory, dashboard, landing page, product page, archive page, single page, Elementor widget, and WordPress plugin work.

## Required controls

Each relevant element should include controls for text, visibility, alignment, width, padding, margin, gap, typography, colors, background, border, radius, shadow, hover state, and desktop/tablet/mobile settings.

## Elements covered

Headings, labels, descriptions, images, icons, cards, boxes, buttons, tags, badges, stats, meta boxes, breadcrumbs, forms, galleries, CTA blocks, empty states, and dynamic data blocks should all have proper controls when used.

## Dynamic widgets

Dynamic widgets should include source, include/exclude, limit, order, filters, empty state, fallback, URL pattern, template page assignment, URL parameter mode, and editor preview item where relevant.

## Safety

Use section-wise widgets where practical. Keep CSS scoped. Keep builds lightweight. Avoid broad theme or WooCommerce changes unless explicitly requested. Keep controls clear enough for a non-coder.

## Delivery checklist

For plugin/widget builds, provide a test package, source update pack, and test report where relevant. Check version, syntax, structure, scoped CSS, and conflict safety. Final lock happens only after live testing and approval.
